# Meeting Prep スキル

## 概要

面談・ミーティング前の情報整理を自動化する。2つのモードで動作:
- **候補者モード**: 採用面談前の候補者ブリーフィング生成
- **クライアントMTGモード**: クライアント定例前のアジェンダ・要注意トピック生成

## モード判定

入力パターンから自動判定:
- `meeting-prep 山田太郎` or `面談準備 候補者ID` → **候補者モード**
- `meeting-prep ClientB` or `MTG準備 クライアント名` → **クライアントMTGモード**
- 不明な場合 → ユーザーに確認

---

## 候補者モード

### 前提条件

| 依存 | パス / 設定 | 必須 |
|------|------------|------|
| 候補者データソース | Config `CANDIDATE_DATA_SOURCE` | YES |
| 面談記録 | Config `MEETING_NOTES_DIR` | あれば参照 |

### 実行フロー

1. 候補者名 or IDを受け取る
2. Config `CANDIDATE_DATA_SOURCE` で指定されたデータソースから候補者情報を検索
3. 取得項目:
   - 経歴・現職・スキル
   - 過去のやり取り履歴（あれば）
   - 評価ランク（あれば）
4. Config `MEETING_NOTES_DIR` から過去の面談記録をGrep
5. ブリーフィング生成:

```markdown
# 候補者ブリーフィング: [氏名]

## 基本情報
- 現職: [会社名] / [役職]
- 経験年数: X年
- スキル: [主要スキル]
- 評価ランク: [S/A/B/C]

## やり取り経緯
- 初回コンタクト: YYYY-MM-DD
- 返信: YYYY-MM-DD「[返信要約]」
- 面談設定: YYYY-MM-DD XX:XX

## 注意点
- [過去の面談でのフィードバック]
- [懸念事項]

## 質問候補
1. [ポジションフィット確認の質問]
2. [動機・転職理由の深堀り]
3. [カルチャーフィット確認]
```

## クライアントMTGモード

### 前提条件

| 依存 | パス / 設定 | 必須 |
|------|------------|------|
| クライアントフォルダ | Config `CLIENT_BASE`/{client}/ | YES |
| 戦略ドキュメント | Config `CLIENT_BASE`/{client}/Config `STRATEGY_DIR` | YES |
| 議事録 | Config `CLIENT_BASE`/{client}/Config `MEETING_NOTES_DIR` | あれば参照 |
| インシデント記録 | Config `CLIENT_BASE`/{client}/Config `INCIDENT_DIR` | あれば参照 |

### 実行フロー

1. クライアント名を受け取る
2. `{CLIENT_BASE}/{client}/{STRATEGY_DIR}` から現在のKGI/戦略を全読み込み
3. `{CLIENT_BASE}/{client}/{MEETING_NOTES_DIR}` から直近3件の議事録を要約
4. `{CLIENT_BASE}/{client}/{INCIDENT_DIR}` から未解決 or 直近のインシデントを抽出
5. プロジェクトのタスク管理ファイルからクライアント関連タスクの進捗を収集
6. アジェンダ + ブリーフィング生成:

```markdown
# MTGブリーフィング: [クライアント名] 定例（YYYY-MM-DD）

## 前回MTGからの進捗
- [前回議事録のアクションアイテム状況]

## KGI進捗サマリー
| KGI | 目標 | 現状 | 進捗率 |
|-----|------|------|--------|
| KGI1 | [目標] | [現状] | XX% |
| KGI2 | [目標] | [現状] | XX% |

## 要注意トピック
- [未解決インシデント]
- [ブロッカーになっているタスク]
- [期限が迫っているアイテム]

## 提案アジェンダ
1. 前回アクションアイテム確認（5min）
2. KGI進捗共有（10min）
3. [要注意トピック]の対応方針（10min）
4. 来週の計画すり合わせ（5min）

## 報告用データ
- [必要な数値・グラフの参照先]
```

## エラーハンドリング

| エラー | 対処 |
|--------|------|
| 候補者が見つからない | 「データソースに該当者なし。名前を確認してください」 |
| クライアントフォルダ不在 | 「{CLIENT_BASE}/{name}/ が存在しません。正しいクライアント名は？」 |
| データソースAPI認証失敗 | ファイルベースの情報のみで生成 |
| 議事録が0件 | 「過去の議事録なし。初回MTGとして準備します」 |

## 設計原則

| 原則 | 出典 | 適用 |
|------|------|------|
| モード自動判定 | UX原則 | 入力パターンから候補者/クライアントMTGを自動判定。不明時のみ確認 |
| 情報源の優先順位 | Graceful Degradation | API > ローカルファイルの順。上位が使えなければ下位で代替 |
| 面談前24時間以内に実行 | 鮮度原則 | ブリーフィングは鮮度が命。古いデータで準備しない |

## Config

| カテゴリ | キー | デフォルト値 | 説明 |
|---------|------|------------|------|
| パス | CLIENT_BASE | `clients/` | クライアントディレクトリのルート |
| パス | MEETING_NOTES_DIR | `meeting_notes/` | 議事録格納先（クライアント配下） |
| パス | STRATEGY_DIR | `strategy/` | 戦略ドキュメント格納先 |
| パス | INCIDENT_DIR | `incidents/` | インシデント格納先 |
| パス | CANDIDATE_DATA_SOURCE | `candidates/` | 候補者データのルートパスまたはAPI設定 |
| API | SA_EMAIL | 環境変数 | Google Sheets サービスアカウント（任意） |
| API | SPREADSHEET_ID | 環境変数 | スカウトシートのID（任意） |
| 設定 | RECENT_NOTES_COUNT | `3` | 参照する直近議事録の件数 |

## セキュリティ

| 項目 | ルール |
|------|--------|
| API認証情報 | 環境変数から取得。ログ出力禁止 |
| PII（候補者個人情報） | 氏名・メール・電話はブリーフィング内のみに記載。ログ・コミットメッセージに含めない |

## BLOCKINGゲート

| Step | 失敗条件 | 動作 |
|------|---------|------|
| モード判定 | 候補者名もクライアント名も特定できない | STOP + 「候補者名 or クライアント名を指定してください」 |
| 候補者モード Step 2 | データソース認証失敗 + ローカルファイルも存在しない | STOP + 「候補者情報のソースがありません」 |
| クライアントMTG Step 1 | クライアントディレクトリが存在しない | STOP + 「{CLIENT_BASE}/{name}/ が存在しません」 |
| クライアントMTG Step 2 | strategy/ に戦略ドキュメントが1件もない | STOP + 「KGI/戦略が未定義です。先に設定してください」 |

## エスカレーション

| 状況 | 対応 |
|------|------|
| 候補者が複数マッチ（同姓同名） | ユーザーに確認: 「{名前}が複数見つかりました。どの候補者ですか？ [一覧表示]」 |
| 未解決CRITICALインシデントがある状態でMTG | ユーザーに確認: 「CRITICALインシデントが未解決です。MTGアジェンダの冒頭に入れますか？」 |
| 候補者ランクがNG判定の面談準備 | ユーザーに確認: 「NG判定の候補者です。面談を進めますか？」 |

## やらないこと

- 面談の実施・議事録の作成（準備資料の生成まで。面談自体はユーザーが実施）
- 候補者への連絡・スカウト送信
- クライアント戦略の変更提案（現状のKGI/戦略を整理して表示するのみ）
- 過去の面談評価の変更（過去データは読み取り専用）

## 汎用性（Portability）

このスキルは **フレームワーク（汎用）+ Config（組織固有）** の構造。
他社は Config セクションの `CLIENT_BASE`, `CANDIDATE_DATA_SOURCE` 等を自社環境に変更するだけで使える。
候補者ブリーフィング・クライアントMTGアジェンダのフレームワーク（テンプレート構造・情報収集順序）はプロジェクト非依存。

企業固有の値は全てConfigセクションに集約済み。
